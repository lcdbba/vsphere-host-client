Migrieren einer virtuellen Maschine

Sie können virtuelle Maschinen per Migration von einem Host auf einen anderen verschieben. Es stehen folgende Migrationsoptionen zur Verfügung:

vSphere vMotion (Live-Migration)
vSphere vMotion verschiebt eine ausgeführte virtuelle Maschine von einem Host auf einen anderen. Wenn die virtuelle Maschine eingeschaltet ist und Speicher- und Netzwerkoptionen verwendet, die auf dem neuen Host verfügbar sind, kann sie ohne Unterbrechung von Anwendungen oder des Gastbetriebssystems verschoben werden.

Cold-Migration
Bei einer Cold-Migration ("kalten Migration") werden virtuelle Maschinen in ausgeschaltetem Zustand von einem Host auf einen anderen verschoben. Diese Art der Migration kann durchgeführt werden, indem die virtuelle Maschine per Drag & Drop innerhalb des gleichen Datencenters von einem Host auf einen anderen gezogen wird.

Sie können virtuelle Maschinen zwischen Hosts im gleichen Datencenter verschieben, nicht jedoch von einem Datencenter in ein anderes. Sie können virtuelle Maschinen entweder manuell verschieben oder eine geplante Aufgabe für die Migration einrichten.
