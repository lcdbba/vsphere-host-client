Qu'est-ce que l'architecture de commutation distribuée vSphere ?

La gestion de réseau avec les commutateurs distribués vSphere est pilotée au niveau du centre de données. Un commutateur distribué vSphere existe comme objet dans le centre de données qui fait office de commutateur virtuel sur tous les hôtes associés, permettant aux machines virtuelles de conserver une configuration réseau cohérente lors de leur migration sur les hôtes.

Les liaisons montantes connectent le commutateur distribué vSphere aux cartes réseau physiques sur les hôtes associés. Chaque hôte peut connecter un adaptateur réseau physique à un commutateur distribué vSphere pour chaque port de liaison montante sur le commutateur distribué vSphere.

Les cartes NIC des machines virtuelles se connectent aux commutateurs distribués vSphere via des groupes de ports distribués qui agrègent plusieurs ports sous une configuration commune et fournissent un point d'ancrage stable pour les machines virtuelles qui se connectent à un réseau nommé.
