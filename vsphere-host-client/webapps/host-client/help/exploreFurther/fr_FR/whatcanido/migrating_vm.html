Migration d'une machine virtuelle

Vous pouvez déplacer des machines virtuelles en utilisant la migration, processus de déplacement d'une machine virtuelle d'un hôte à un autre. Les options de migration sont les suivantes :

vSphere vMotion (migration en temps réel)
vSphere vMotion transfère une machine virtuelle active d'un hôte vers un autre. Si la machine virtuelle est activée et utilise des options de stockage et de réseau disponibles chez le nouvel hôte, elle peut être déplacée sans arrêter d'applications ni perturber le système d'exploitation client.

Migration à froid
La migration à froid déplace une machine virtuelle désactivée d'un hôte à un autre. La migration peut être exécutée en glissant la machine virtuelle d'un hôte à un autre dans le même centre de données.

Vous pouvez déplacer des machines virtuelles entre les hôtes d'un même centre de données, mais vous ne pouvez pas en déplacer d'un centre de données à un autre. Vous pouvez déplacer les machines virtuelles manuellement ou installer une tâche planifiée pour exécuter la migration.
