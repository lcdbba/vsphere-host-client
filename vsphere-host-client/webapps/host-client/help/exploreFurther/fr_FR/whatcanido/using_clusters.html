Utilisation des clusters

Employez les clusters pour maximiser l'utilisation de l'hôte et pour assurer une haute disponibilité aux machines virtuelles. Les clusters distribuent les ressources entre les hôtes et gèrent l'équilibrage de charge. En utilisant des clusters, vous pouvez activer les solutions VMware suivantes :

  • vSphere Distributed Resource Scheduler (DRS)
    vSphere DRS équilibre continuellement les charges de travail des machines virtuelles sur vos hôtes ESX/ESXi. Quand l'activité d'une machine virtuelle exige plus de ressources en CPU ou en mémoire que ce que l'hôte sur lequel elle est exécutée peut fournir, le système vCenter Server déclenche des migrations vMotion pour équilibrer la charge de travail.
  • vSphere Haute Disponibilité (HA)
    vSphere HA permet aux machines virtuelles fonctionnant sur des hôtes ESX/ESXi de récupérer automatiquement en cas d'une panne d'hôte. Quand un hôte tombe en panne, toutes les machines virtuelle associées sont immédiatement redémarrées sur d'autres hôtes du cluster.
  • La fonction VMware Enhanced vMotion Compatibility (EVC)
    aide à assurer la compatibilité de vMotion entre les hôtes d'un cluster en garantissant que tous les hôtes du cluster offrent les mêmes fonctionnalités de CPU aux machines virtuelles.

Vous pouvez raffiner encore l'utilisation du cluster en créant des pools de ressources pour partitionner hiérarchiquement les ressources CPU et mémoire disponibles dans un cluster.
