クラスタの使用

ホストの使用率を最大にし、仮想マシンに高可用性を提供するためにクラスタを使用します。クラスタはホスト間にリソースを分散させてロード バランシングを管理します。クラスタを使用すると、次の VMware ソリューションを使用することができます。

  • vSphere DRS (Distributed Resource Scheduler)
    vSphere DRS は、ESX/ESXi ホスト間で、継続的に仮想マシンのワークロードのバランスをとります。仮想マシン アクティビティで、CPU またはメモリ リソースが、その仮想マシンを実行しているホストが提供できるよりも多く必要な場合、ワーク ロードのバランスをとるために、vCenter Server システムによって vMotion 移行が開始されます。
  • vSphere HA (High Availability)
    vSphere HA を使用すると、ESX/ESXi ホストで実行している仮想マシンは、ホストの障害時に自動的に復旧できます。ホストに障害が発生すると、関連付けられた仮想マシンはすべて、クラスタ内のほかのホストでただちに再起動されます。
  • VMware EVC （Enhanced vMotion Compatibility）
    EVC は、クラスタ内のすべてのホストが確実に同じ CPU 機能セットを仮想マシンに提供することで、1 つのクラスタ内のホスト間で vMotion の互換性を確保するのに役立ちます。

リソース プールを作成し、クラスタの利用可能な CPU およびメモリ リソースを階層的に分割することによって、クラスタの使用率をさらにあげることができます。
