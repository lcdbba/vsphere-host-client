클러스터 사용

호스트 활용도를 극대화하고 가상 시스템에 대한 고가용성을 제공하려면 클러스터를 사용합니다. 클러스터는 호스트 전체에 리소스를 분산하고 로드 밸런싱을 관리합니다. 클러스터를 사용하면 다음 VMware 솔루션을 사용하도록 설정할 수 있습니다.

  • VMware DRS(Distributed Resource Scheduler)
    vSphere DRS는 ESX/ESXi 호스트 전체에서 가상 시스템 워크로드 밸런싱을 수행합니다. 가상 시스템 작업이 가상 시스템이 실행되는 호스트에서 제공할 수 있는 것보다 많은 CPU 또는 메모리 리소스를 필요로 할 경우 vCenter Server 시스템에서 워크로드 밸런싱을 수행하도록 vMotion 마이그레이션을 트리거합니다.
  • vSphere HA(High Availability)
    vSphere HA를 사용하면 ESX/ESXi 호스트에서 실행되고 있는 가상 시스템이 자동으로 호스트 장애에서 복구됩니다. 호스트에 장애가 발생할 경우 연관된 모든 가상 시스템이 클러스터의 다른 호스트에서 즉시 다시 시작됩니다.
  • VMware EVC(향상된 vMotion 호환성)
    EVC는 클러스터 내의 모든 호스트가 가상 시스템에 동일한 CPU 기능 집합을 제공하도록 하여 클러스터 내의 호스트 간에 vMotion 호환성을 보장합니다.

클러스터의 사용 가능한 CPU 및 메모리 리소스를 계층적으로 파티셔닝하는 리소스 풀을 생성하여 클러스터 사용을 보다 구체화할 수 있습니다.
